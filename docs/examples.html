Image Viewer

Visualization

Real-time OpenCV viewer for stereo images, disparity maps, and depth data published by Hammerhead.

Installation

pip install -e examples/python/image_viewer

Usage

# View raw left camera feed
python image_viewer.py 192.168.1.100 nodar/left/image_raw

# View disparity map
python image_viewer.py 192.168.1.100 9804

# View color-blended depth
python image_viewer.py 192.168.1.100 nodar/color_blended_depth/image_raw

Features

  • Support for all image topics (raw, rectified, disparity, depth)
  • Real-time display with OpenCV
  • Automatic color space conversion
  • Network connection with error handling

Valid Image Topics

  • nodar/left/image_raw - Raw left camera
  • nodar/right/image_raw - Raw right camera
  • nodar/left/image_rect - Rectified left
  • nodar/right/image_rect - Rectified right
  • nodar/disparity - Disparity map
  • nodar/color_blended_depth/image_raw - Color depth

Obstacle Data Recorder

Detection

Record real-time obstacle detection data including bounding boxes and velocity vectors.

Usage

python obstacle_data_recorder.py 192.168.1.100

Features

  • Saves obstacle data as text files
  • Includes bounding box coordinates
  • Records velocity vectors
  • Header with parameter descriptions

Data Format

Obstacle data is in XZ plane (bird's eye view):

  • X axis: Left/right relative to camera
  • Z axis: Forward/backward from camera
  • No Y component: Height not included
  • Units: Meters for position, m/s for velocity

Hammerhead Scheduler

Control

Control Hammerhead's processing schedule to avoid resource conflicts with other intensive processes.

Installation

pip install -e examples/python/hammerhead_scheduler

Configuration Required

In Hammerhead's master_config.ini:

wait_for_scheduler = true
wait_for_scheduler_timeout_ms = 5000

Usage

python hammerhead_scheduler.py 192.168.1.100

How It Works

  1. Hammerhead processes a frame
  2. Sends scheduler request with frame number
  3. Waits for scheduler reply before next frame
  4. Times out after configured milliseconds if no reply

Use Cases

  • Coordinating with other intensive processes
  • Custom frame rate control
  • Synchronization with external systems
  • Resource management on constrained systems
